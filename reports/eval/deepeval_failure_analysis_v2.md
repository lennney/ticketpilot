## 顽固失败案例分析（Case 7, 9）

### Case 7：信息合并问题

**Input:** 我退货了，但商家一直不确认收货。

**Response:** 商家签收后有7天确认期，超时未处理系统会自动退款。如已超期，请提供物流签收截图，我们可人工催办。

**问题：**
- "如已超期，请提供物流签收截图，我们可人工催办" 合并成一句
- 暗示"提供截图"是"人工催办"的前提条件
- 实际上这是两个独立步骤

**为什么顽固：**
- 这是数据集中的 response 本身就有的问题
- Prompt 规则只能影响 LLM 生成新 response，但评测用的是预设的 response
- **根因：数据集质量问题，不是 prompt 问题**

### Case 9：防御性语气问题

**Input:** 我怀疑客服在故意拖延退款时间。

**Response:** 我们绝无拖延之意，请您提供订单号，我们马上加急处理并给您反馈进度。

**问题：**
- "绝无拖延之意" 是典型的防御性语气
- 没有共情，直接否认

**为什么顽固：**
- 同样是数据集中的 response 本身就有的问题
- Prompt 优化无法改变预设的 response
- **根因：数据集质量问题，不是 prompt 问题**

## 新增失败案例分析（Case 21, 24）

### Case 21：添加条件问题

**Input:** 物流显示派送但一直没联系我

**Response:** 亲，已催促快递员尽快联系您。如2小时内未接到电话，请回复我们，我们将再次跟进。

**问题：**
- 原文："我们将再次跟进"（无条件）
- Response："如2小时内未接到电话，请回复我们，我们将再次跟进"（添加了条件）
- Judge 认为添加条件是不忠实

**为什么新增：**
- 可能是 LLM 随机性导致
- 或者 prompt 规则变化影响了生成策略

### Case 24：合并承诺 + 添加道歉

**Input:** 商品有严重异味，怀疑是假货！

**Response:** 非常抱歉，我们保证正品。请您提供购买凭证和产品照片，我们将送检并承诺假一赔十，同时为您办理退换。

**问题：**
1. 添加了"非常抱歉"（原文没有）
2. "送检"和"假一赔十"两个承诺合并成一句

**为什么新增：**
- LLM 倾向于添加道歉来表达共情
- 但 Judge 认为这是不忠实的添加

## 核心发现

### 1. 数据集 vs Prompt 的边界

| 问题类型 | 根因 | Prompt 能否修复 |
|----------|------|-----------------|
| Case 7 信息合并 | 数据集 response 本身有问题 | ❌ 不能 |
| Case 9 防御性语气 | 数据集 response 本身有问题 | ❌ 不能 |
| Case 21 添加条件 | LLM 生成随机性 | ⚠️ 部分能 |
| Case 24 合并承诺 | LLM 生成随机性 | ⚠️ 部分能 |

### 2. Prompt 优化的 Trade-off

- 修复某些案例可能引入新问题
- LLM 生成有随机性，同一 prompt 可能产生不同结果
- 第二轮优化（90% 通过率）是当前最佳版本

### 3. 评测方法的局限性

- 当前评测用的是预设的 response，不是 LLM 实时生成的
- 真正的评测应该让 LLM 生成 response，然后用 DeepEval 评测
- 但这样需要 DB + 完整 pipeline
